Actions let you orchestrate complex transformations of your files after your repository is pulled down. Each action is an object added to the `actions` array in your `.bluprintrc` file.

## How actions work

Actions are defined as objects in the `actions` array and run sequentially in the order they appear:

```json
{
  "bluprint": "^0.0.1",
  "name": "My bluprint",
  "category": "",
  "actions": [{
    "action": "prompt"
    // Runs first...
  }, {
    "action": "render"
    // Runs second...
  }, {
    "action": "log"
    // Runs last...
  }]
}
```

You can define as many actions as you like. Check out the [example bluprint](https://github.com/reuters-graphics/bluprint/blob/master/docs/example.md) to see what you can do with actions.

## Available actions

### prompt

Ask users for information that can be used in subsequent actions.

```json
{
  "action": "prompt",
  "questions": [{
    "type": "text",
    "name": "projectName",
    "message": "What should we call this project?"
  }]
}
```

**Properties:**
- `questions`: Array of [prompts.js](https://github.com/terkelg/prompts) questions

The `name` of each question becomes available to all subsequent actions that support templating, such as `render`, `log`, `move`, and `regexreplace`.

### render

Pass files through a templating engine with custom context.

```json
{
  "action": "render",
  "engine": "mustache",
  "files": ["README.md", "package.json"],
  "questions": [],
  "context": {
    "copyrightYear": "2020"
  }
}
```

**Properties:**
- `engine`: Templating engine to use (`"mustache"` or `"ejs"`)
- `files`: Array of files to render
- `questions`: Optional array of [prompts.js](https://github.com/terkelg/prompts) questions (context only available in this action)
- `context`: Optional object of additional context

Context includes answers from previous `prompt` actions, plus utility functions from the [string](https://www.npmjs.com/package/string) package: `camelize`, `capitalize`, `dasherize`, `humanize`, `latinise`, `slugify`, `titleCase`, and `underscore`.

**EJS example:**
```ejs
<%= slugify(myVariable) %>
```

**Mustache example:**
```mustache
{{#slugify}}{{myVariable}}{{/slugify}}
```

### log

Display messages to users during scaffolding.

```json
{
  "action": "log",
  "msg": "Finished creating your project!"
}
```

**Properties:**
- `msg`: Message string

Messages support [chalk](https://github.com/chalk/chalk) tagged template syntax for color:
```json
"msg": "Welcome to your {green new} project!"
```

And [mustache](https://mustache.github.io/) syntax for dynamic values:
```json
"msg": "Scaffolded your new project named {{ projectName }}!"
```

### move

Move or rename files and directories.

```json
{
  "action": "move",
  "paths": [
    ["from/", "to/"],
    ["moveme.md", "moved.md"]
  ]
}
```

**Properties:**
- `paths`: Array of `[source, destination]` pairs

Supports mustache template syntax in destination paths:
```json
["moveme/code.js", "{{ someAnswer }}/code.js"]
```

### copy

Copy files or directories.

```json
{
  "action": "copy",
  "paths": [
    ["from/", "to/"],
    ["copyme.md", "copied.md"]
  ]
}
```

**Properties:**
- `paths`: Array of `[source, destination]` pairs

Supports mustache template syntax in destination paths:
```json
["copyme/code.js", "{{ someAnswer }}/code.js"]
```

### remove

Delete files or directories.

```json
{
  "action": "remove",
  "paths": [
    "dir/*",
    "README.md"
  ]
}
```

**Properties:**
- `paths`: Array of glob strings relative to the project root

### regexreplace

Replace text in files using regular expressions.

```json
{
  "action": "regexreplace",
  "files": ["README.md"],
  "replace": [
    ["color", "colour"],
    ["([0-9]{2})\/([0-9]{2})\/([0-9]{4})", "$2.$1.$3", "g"]
  ]
}
```

**Properties:**
- `files`: Array of files to modify
- `replace`: Array of `[pattern, replacement, flags?]` arrays
  - `pattern`: Regular expression string
  - `replacement`: Replacement string (supports regex capture groups and mustache syntax)
  - `flags`: Optional regex flags (defaults to `gm`)

Example with mustache:
```json
["^Name: .+$", "Name: {{ userName }}"]
```

### execute

Run shell commands.

```json
{
  "action": "execute",
  "cmds": [
    ["yarn"],
    ["git", ["init"]]
  ],
  "silent": true
}
```

**Properties:**
- `cmds`: Array of command arrays. Each inner array has:
  - First item: command string
  - Second item (optional): array of arguments
- `silent`: Optional boolean to suppress command output

## Default context

The following variables are automatically available to actions that support templating (`log`, `move`, `regexreplace`, `render`):

- `year`: Full year at runtime (e.g., `2022`)
- `month`: Zero-padded month (e.g., `02`)
- `day`: Zero-padded date (e.g., `07`)
- `dirname`: Name of the parent directory where bluprint is executed (e.g., `my-project-folder`)

Example usage:

```json
{
  "action": "regexreplace",
  "files": ["my-file.txt"],
  "replace": [
    ["YYYY", "{{ year }}"],
    ["project-name", "{{ dirname }}"]
  ]
}
```

## Conditional actions

All actions can be conditionally run based on previous prompt answers using a `condition` key:

```json
[
  {
    "action": "prompt",
    "questions": [{
      "type": "text",
      "name": "userName",
      "message": "What's your name?"
    }]
  },
  {
    "action": "log",
    "msg": "Hi, Jon!",
    "condition": ["userName", "Jon"]
  }
]
```

The condition array contains:
- First item: [string object path](https://lodash.com/docs/4.17.15#get) of the prompt variable
- Second item: expected value

Actions that fail the condition test are skipped.

### Multiple conditions

Test multiple values at once:

```json
{
  "condition": [
    ["myPromptVar", "some value"],
    ["myOtherVar", true]
  ]
}
```

### Conditioning on parts

Use the special `bluprintPart` context variable to run actions based on selected parts:

```json
{
  "condition": ["bluprintPart", "Config files"]
}
```

To run only when the whole bluprint is used:

```json
{
  "condition": ["bluprintPart", null]
}
```
