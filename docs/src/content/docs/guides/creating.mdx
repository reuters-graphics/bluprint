---
title: Creating a bluprint
description: Learn how to create and configure your own bluprint template
---

A bluprint is any GitHub repository you want to use as a template to scaffold out a new project. Creating one is simple: just add a `.bluprintrc` JSON file to the root of your repository.

## Creating the configuration file

Use the `new` command in the root of the project you'd like to templatize:

```bash
bluprint new
```

This creates a `.bluprintrc` file with the basic structure:

```json
{
  "bluprint": "^0.0.1",
  "name": "My bluprint",
  "category": "",
  "actions": []
}
```

## Configuration options

### bluprint (required)

The version of bluprint your template is compatible with. Uses semantic versioning.

```json
{
  "bluprint": "^0.0.1"
}
```

### name (required)

A human-readable name for your bluprint that will be displayed in the CLI.

```json
{
  "name": "My React Component Template"
}
```

### category (optional)

A category to organize your bluprints in the CLI. Similar bluprints can share the same category.

```json
{
  "category": "React Components"
}
```

### actions (optional)

An array of actions to process your bluprint files after scaffolding. Actions run in sequence and can transform files, prompt for input, execute commands, and more.

```json
{
  "actions": [
    {
      "action": "prompt",
      "questions": [{
        "type": "text",
        "name": "componentName",
        "message": "What should we call this component?"
      }]
    },
    {
      "action": "render",
      "engine": "mustache",
      "files": ["README.md"]
    }
  ]
}
```

Learn more about actions in the [Actions guide](/guides/actions).

### parts (optional)

An object defining reusable parts of your bluprint that users can apply individually. This is useful for updating specific files in existing projects.

```json
{
  "parts": {
    "Config files": [
      "config/*",
      ".tasksrc"
    ],
    "JS components": [
      "src/js/components/**/*"
    ]
  }
}
```

Learn more in the [Parts guide](/guides/parts).

### mergeJson (optional)

When set to `true`, attempts to merge existing JSON files with JSON files from your bluprint using lodash merge.

```json
{
  "mergeJson": true
}
```

This is particularly useful when updating dependencies in `package.json` or other configuration files. Properties in the bluprint's JSON file will take precedence.

## Publishing your bluprint

Once you've created your `.bluprintrc` file:

1. Commit the file to your repository
2. Push to GitHub
3. The bluprint is now ready to use!

No special publishing step is required - the CLI pulls directly from GitHub.

Check out the [example bluprint](https://github.com/reuters-graphics/bluprint/blob/master/docs/example.md) in the repository to see a complete configuration with multiple actions and parts.
