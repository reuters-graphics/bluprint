This page provides a complete reference for all available actions in bluprint. For a guide on using actions, see the [Actions guide](/guides/actions).

## Common properties

All actions support these properties:

### condition

Conditionally run an action based on a previous prompt value.

**Type:** `Array`

**Format:**
- Single condition: `[variablePath, expectedValue]`
- Multiple conditions: `[[path1, value1], [path2, value2]]`

**Examples:**

```json
{
  "condition": ["userName", "Jon"]
}
```

```json
{
  "condition": [
    ["useTypeScript", true],
    ["framework", "React"]
  ]
}
```

Special conditions:
- Test for selected part: `["bluprintPart", "Config files"]`
- Test for full bluprint: `["bluprintPart", null]`

## copy

Copy files or directories to new locations.

### Properties

#### paths (required)

Array of `[source, destination]` pairs.

**Type:** `Array<[string, string]>`

**Templating:** Destination paths support mustache template syntax

**Examples:**

```json
{
  "action": "copy",
  "paths": [
    ["src/template.js", "src/component.js"],
    ["config/", "backup/config/"]
  ]
}
```

With templating:

```json
{
  "action": "copy",
  "paths": [
    ["template/", "{{ projectName }}/"],
    ["README.md", "{{ dirname }}/README.md"]
  ]
}
```

## execute

Execute shell commands.

### Properties

#### cmds (required)

Array of commands to execute. Each command is an array where:
- First item: command string
- Second item (optional): array of arguments

**Type:** `Array<[string] | [string, string[]]>`

**Examples:**

```json
{
  "action": "execute",
  "cmds": [
    ["yarn"],
    ["git", ["init"]],
    ["npm", ["install", "--save-dev", "eslint"]]
  ]
}
```

#### silent

Suppress command output.

**Type:** `boolean`

**Default:** `false`

**Example:**

```json
{
  "action": "execute",
  "cmds": [["yarn", ["install"]]],
  "silent": true
}
```

## log

Display messages to users during bluprint execution.

### Properties

#### msg (required)

Message to display.

**Type:** `string`

**Templating:** Supports chalk color syntax and mustache template syntax

**Chalk syntax:**

```json
{
  "action": "log",
  "msg": "Your {green new} project is {blue ready}!"
}
```

Available colors: `black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white`, `gray`, `grey`, `blackBright`, `redBright`, `greenBright`, `yellowBright`, `blueBright`, `magentaBright`, `cyanBright`, `whiteBright`

**Mustache syntax:**

```json
{
  "action": "log",
  "msg": "Created project {{ projectName }} in {{ year }}"
}
```

**Combined:**

```json
{
  "action": "log",
  "msg": "{green Success!} Created {{ projectName }}"
}
```

## move

Move or rename files and directories.

### Properties

#### paths (required)

Array of `[source, destination]` pairs.

**Type:** `Array<[string, string]>`

**Templating:** Destination paths support mustache template syntax

**Examples:**

```json
{
  "action": "move",
  "paths": [
    ["old-name.js", "new-name.js"],
    ["src/temp/", "src/final/"]
  ]
}
```

With templating:

```json
{
  "action": "move",
  "paths": [
    ["template.js", "{{ componentName }}.js"],
    ["generic/", "{{ dirname }}/"]
  ]
}
```

## prompt

Ask users for input during scaffolding.

### Properties

#### questions (required)

Array of prompts.js question objects.

**Type:** `Array<Object>`

**Question object properties:**

- `type`: Type of prompt (see below)
- `name`: Variable name for the answer
- `message`: Question to display
- `initial`: Default value (optional)
- `choices`: Array of choices for select/multiselect (optional)

**Prompt types:**

- `text`: Single-line text input
- `number`: Numeric input
- `confirm`: Yes/no confirmation
- `select`: Choose one from a list
- `multiselect`: Choose multiple from a list
- `toggle`: Binary choice
- `date`: Date picker

**Examples:**

Basic text prompt:
```json
{
  "action": "prompt",
  "questions": [{
    "type": "text",
    "name": "projectName",
    "message": "What should we call this project?"
  }]
}
```

Multiple questions:
```json
{
  "action": "prompt",
  "questions": [
    {
      "type": "text",
      "name": "authorName",
      "message": "Author name:",
      "initial": "Anonymous"
    },
    {
      "type": "confirm",
      "name": "useTypeScript",
      "message": "Use TypeScript?",
      "initial": true
    },
    {
      "type": "select",
      "name": "framework",
      "message": "Choose a framework:",
      "choices": [
        { "title": "React", "value": "react" },
        { "title": "Vue", "value": "vue" },
        { "title": "Svelte", "value": "svelte" }
      ]
    }
  ]
}
```

See [prompts.js documentation](https://github.com/terkelg/prompts) for more details.

## regexreplace

Replace text in files using regular expressions.

### Properties

#### files (required)

Array of file paths to modify.

**Type:** `Array<string>`

**Example:**

```json
{
  "files": ["README.md", "package.json", "src/**/*.js"]
}
```

#### replace (required)

Array of replacement operations. Each operation is an array of:
- `pattern`: Regular expression string
- `replacement`: Replacement string
- `flags`: Optional regex flags (default: `"gm"`)

**Type:** `Array<[string, string] | [string, string, string]>`

**Templating:** Replacement strings support mustache template syntax

**Examples:**

Simple replacement:
```json
{
  "action": "regexreplace",
  "files": ["README.md"],
  "replace": [
    ["color", "colour"]
  ]
}
```

With regex capture groups:
```json
{
  "action": "regexreplace",
  "files": ["dates.txt"],
  "replace": [
    ["([0-9]{2})\/([0-9]{2})\/([0-9]{4})", "$2.$1.$3", "g"]
  ]
}
```

With mustache templating:
```json
{
  "action": "regexreplace",
  "files": ["package.json"],
  "replace": [
    ["\"name\": \".+\"", "\"name\": \"{{ projectName }}\""],
    ["\"author\": \".+\"", "\"author\": \"{{ authorName }}\""]
  ]
}
```

With default context:
```json
{
  "action": "regexreplace",
  "files": ["LICENSE"],
  "replace": [
    ["YYYY", "{{ year }}"],
    ["PROJECT_NAME", "{{ dirname }}"]
  ]
}
```

**Regex flags:**

- `g`: Global (replace all occurrences)
- `i`: Case insensitive
- `m`: Multiline
- `s`: Dot matches newline
- `u`: Unicode
- `y`: Sticky

Default flags are `gm` (global, multiline).

## remove

Delete files or directories.

### Properties

#### paths (required)

Array of glob patterns matching files or directories to remove.

**Type:** `Array<string>`

**Glob patterns:**

- `*`: Match any characters except `/`
- `**`: Match any characters including `/`
- `?`: Match a single character
- `[abc]`: Match any character in the set
- `{a,b}`: Match any of the alternatives

**Examples:**

```json
{
  "action": "remove",
  "paths": [
    "README.md",
    "docs/*",
    "temp/**/*"
  ]
}
```

Remove specific files:
```json
{
  "action": "remove",
  "paths": [
    ".git",
    "package-lock.json",
    "node_modules"
  ]
}
```

Pattern matching:
```json
{
  "action": "remove",
  "paths": [
    "*.log",
    "test/**/*.spec.js",
    "src/**/__tests__"
  ]
}
```

## render

Process files through a templating engine.

### Properties

#### engine (required)

Templating engine to use.

**Type:** `"mustache"` | `"ejs"`

**Example:**

```json
{
  "engine": "mustache"
}
```

#### files (required)

Array of files to render.

**Type:** `Array<string>`

**Example:**

```json
{
  "files": ["README.md", "package.json", "src/config.js"]
}
```

#### questions

Additional prompts for this action only. Answers are only available as context within this render action.

**Type:** `Array<Object>`

**Example:**

```json
{
  "action": "render",
  "engine": "mustache",
  "files": ["config.js"],
  "questions": [{
    "type": "text",
    "name": "apiKey",
    "message": "Enter your API key:"
  }]
}
```

#### context

Additional static context to pass to templates.

**Type:** `Object`

**Example:**

```json
{
  "action": "render",
  "engine": "ejs",
  "files": ["LICENSE"],
  "context": {
    "copyrightYear": "2020",
    "license": "MIT"
  }
}
```

### Available context

Templates have access to:

1. **Previous prompt answers**: Any answers from previous `prompt` actions
2. **Default context**: `year`, `month`, `day`, `dirname`
3. **questions**: Answers from this action's questions (if provided)
4. **context**: Static context from this action (if provided)
5. **Utility functions**: String manipulation functions

### Utility functions

Both EJS and Mustache templates have access to these string utility functions:

- `camelize`: Convert to camelCase
- `capitalize`: Capitalize first letter
- `dasherize`: Convert to dash-case
- `humanize`: Convert to human readable
- `latinise`: Remove accents
- `slugify`: Convert to URL-safe slug
- `titleCase`: Convert to Title Case
- `underscore`: Convert to snake_case

**EJS usage:**

```ejs
<%= slugify(projectName) %>
<%= capitalize(authorName) %>
```

**Mustache usage:**

```mustache
{{#slugify}}{{projectName}}{{/slugify}}
{{#capitalize}}{{authorName}}{{/capitalize}}
```

### Template examples

**Mustache:**

```mustache
# {{ projectName }}

Created by {{ authorName }} in {{ year }}.

## Slug

{{#slugify}}{{projectName}}{{/slugify}}
```

**EJS:**

```ejs
# <%= projectName %>

Created by <%= authorName %> in <%= year %>.

## Slug

<%= slugify(projectName) %>

<% if (useTypeScript) { %>
TypeScript is enabled!
<% } %>
```

## Default context

These variables are automatically available to actions that support templating (`log`, `move`, `regexreplace`, `render`):

- `year`: Full year (e.g., `"2023"`)
- `month`: Zero-padded month (e.g., `"02"`)
- `day`: Zero-padded day (e.g., `"07"`)
- `dirname`: Parent directory name (e.g., `"my-project"`)

**Example:**

```json
{
  "action": "render",
  "engine": "mustache",
  "files": ["COPYRIGHT.txt"],
  "context": {}
}
```

In `COPYRIGHT.txt`:
```
Copyright {{ year }} - All rights reserved.
Project: {{ dirname }}
```

## Action execution order

Actions run sequentially in the order defined in the `actions` array:

```json
{
  "actions": [
    {"action": "prompt", "...": "runs first"},
    {"action": "render", "...": "runs second"},
    {"action": "move", "...": "runs third"},
    {"action": "execute", "...": "runs last"}
  ]
}
```

**Important:** Actions that depend on previous actions (e.g., `render` using prompt answers) must come after their dependencies.

## Error handling

If an action fails:
- Execution stops
- An error message is displayed
- The bluprint process exits with a non-zero code

To handle potential errors in `execute` actions, consider adding error handling in your shell commands:

```json
{
  "action": "execute",
  "cmds": [
    ["sh", ["-c", "command || true"]]
  ]
}
```
